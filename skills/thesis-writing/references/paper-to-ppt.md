# 论文转 PPT — Paper to Chinese PPTX Presentation

将科学论文（PDF/预印本/摘要/阅读笔记）转化为中文 PPTX 汇报课件，适用于期刊俱乐部（journal club）、组会（group meeting）、实验室报告（lab meeting）、论文分享、学术讨论等场景。

论文类型覆盖生命科学、医学、化学、材料科学、环境与地球科学、物理学、工程学、计算生物学、AI/方法学论文、综述等多种学科。

## 核心原则：以论文的科学论证为演示主线

默认的幻灯片逻辑应帮助听众依次回答：
1. 为什么这个问题重要？（Why does this problem matter?）
2. 论文针对什么缺口或瓶颈？（What gap or bottleneck does the paper address?）
3. 作者做了什么？（What did the authors do?）
4. 关键证据是什么？（What is the key evidence?）
5. 为什么我们应该相信结果？（Why should we trust the result?）
6. 有什么是新的、可复用的、有广泛意义的？（What is new, reusable, or broadly meaningful?）
7. 边界和开放问题在哪里？（Where are the boundaries and open questions?）

这比照搬论文章节顺序更重要。

## 精简工作模式

**默认执行**：
- 只阅读理解论文论证所需的源材料
- 只提取真正会出现在幻灯片中的图表/表格
- 创建 PPTX 作为首要交付物
- 运行轻量级结构性检查
- 写简短的质量报告

**默认避免**：
- 提取每张图、每页、每张图像、每个表格、所有补充材料
- 完全 OCR（除非正常文本提取失败或 PDF 是扫描版）
- 保存完整原始论文文本（除非需要调试或复用）
- 安装新依赖（当现有工具即可完成任务时）
- 启动 GUI 应用或桌面自动化来生成预览
- 幻灯片全部渲染预览（当无可靠的 headless 渲染器时）

## 工具链策略

使用跨平台 Python 优先栈：
- PyMuPDF：元数据、文本提取、页面渲染、页面裁剪
- Pillow：图表裁剪、缩略图预览
- python-pptx：幻灯片创作和 PPTX 安全编辑
- zipfile + python-pptx 重新打开验证包结构

此栈需能在 macOS、Linux、Windows 上运行。使用 `pathlib` 路径、项目本地输出目录和 Office 安全字体或主题字体。不硬编码 OS 字体路径或平台特定文件位置。

## 工作流

### Step 1. 读取和提取源材料

从论文中提取：
- 标题、作者、期刊/预印本平台、年份、DOI
- 领域和子领域
- 论文类型
- 核心问题和知识缺口
- 主要主张或论点
- 研究设计、工作流、模型、数据集或实验系统
- 关键方法和对照
- 主要结果和量化发现
- 关键图表、表格和图注
- 验证、稳健性、消融或敏感性分析
- 局限性和未解决问题
- 更广泛的科学/临床/技术/环境意义

采用两遍阅读策略：先捕获元数据、摘要、标题、图注和表注；然后根据需要阅读支持幻灯片的 Results 和 Methods 页面。

### Step 2. 论文分类——设计幻灯片逻辑前

识别论文类型（选择最接近的）：
- 发现/机制论文（discovery/mechanism）
- 转化或应用科学论文（translational/applied science）
- 临床或人群研究（clinical/population study）
- 方法/算法/工具论文（methods/algorithm/tool）
- 资源/数据集/图谱论文（resource/dataset/atlas）
- 组学/单细胞/空间/多模态研究
- 材料/化学/工程性能研究
- 环境/生态/地球系统研究
- 基准/评估论文（benchmark/evaluation）
- 综述/观点/评论（review/perspective/commentary）

然后选择最佳演示逻辑：
- `claim-first`（论断先行）：论文有一个强的中心论断
- `question-to-evidence`（问题到证据）：机制和发现论文
- `problem-to-solution`（问题到方案）：方法、工具和工程论文
- `workflow-to-validation`（工作流到验证）：数据集、图谱、组学、基准
- `evidence-map`（证据地图）：综述和观点

### Step 3. 构建中文演示计划

默认长度：12-16 张幻灯片（15-20 分钟报告）。

默认结构：
1. 标题页
2. 研究背景：为什么这个问题重要
3. 知识缺口 / 技术瓶颈
4. 论文核心问题与主张
5. 研究设计 / 技术路线 / 分析框架
6. 关键证据1
7. 关键证据2
8. 关键证据3
9. 验证、对照或稳健性证据
10. 机制模型 / 方法优势 / 综合框架
11. 创新点与可复用价值
12. 局限性与未解决问题
13. 总结与讨论

根据论文类型调整此结构。不要将每篇论文强行塞入同一模板。

### Step 4. 选择图表——作为证据而非装饰

优先携带论文论证的图表：设计/工作流 → 主要证据 → 验证或稳健性 → 机制/模型/综合 → 实践或概念含义。

优先少量可读的关键面板，而非大量不可读的完整图表。标准 10-14 张幻灯片通常选择 4-8 个图表/表格素材。

### Step 5. 提取和准备图表素材

- 仅对所选的图表页面渲染高分辨率图像
- 当完整图表过于密集时裁切相关面板
- 保持原始数据可视化不变
- 图片保存于 `output/assets/figures/`
- 表格和简单量化比较优先使用可编辑的 PPT 原生表格/图表

### Step 6. 逐张幻灯片编写内容

每张幻灯片撰写：
- 中文标题（结论式标题优先：如"PathAgent主动识别信息不足并补充证据"，而非"Case Study"或"Figure 3"）
- 幻灯片目的
- 建议布局
- 3-4 条简洁中文要点
- 精选的图表或表格素材及其编号
- 中文图表标题和解读
- 一条核心要点（takeaway）
- 中文演讲者备注

每张幻灯片只讲一个要点。结果幻灯片应回答：此图展示了什么？为什么对论文主张重要？听众看完后应相信什么？

### 演讲者备注

应实用而简洁。当幻灯片内容本身已自明时，不要为每张幻灯片写长篇叙述。

### 版面布局规则

不要默认固定的 50/50 左右分割。布局应从图表的宽高比、密度和论证角色出发：
- 复杂宽幅图表 → 全宽或接近全宽
- 纵向图表 → 高图+窄文字栏
- 横向宽裕图表 → 上下堆叠（上方论点+下方视觉）
- 一侧明显主导时 → 非对称分割（70/30、75/25、65/35）
- 仅有少量标注时 → 紧凑的"视觉+标注"布局
- 等权 1:1 作为例外，非默认

## Step 7. 构建 PPTX

使用 python-pptx 作为默认创作工具。PPTX 应：
- 默认使用 16:9 宽屏布局
- 包含精选的原始图表
- 使用中文标题、要点、图注和演讲者备注
- 图表幻灯片包含来源标签（如 `Source: Fig. 2b, Nature, 2024`）
- 保持幻灯片文字简洁可读
- 视觉可用时避免纯文字结果幻灯片
- 保持一致的排版、间距、标题、图注和章节过渡

## Step 8. 渲染、检查和修订

若 headless 渲染器可用，渲染预览并检查：缺失图像、变形或低分辨率图表、不可读面板、文字溢出、标题/要点/图表重叠、过度要点密度、幻灯片顺序错误、缺少来源标签、缺失或无用的演讲者备注。

若无可靠渲染器，进行轻量级验证：重新打开 PPTX 检查幻灯片数量、嵌入媒体数量、演讲者备注存在性和形状边界。

## 风格规则

- 干净的白色或极浅背景
- 深色可读文字
- 一到两种柔和的强调色
- 紧凑但不拥挤的布局
- 图表优先的结果幻灯片
- 简洁的图注
- 无装饰性素材照片、装饰性渐变、夸张的市场风格章节页

## 论文类型专属适配

### 发现/机制论文 → question-to-evidence
现象与重要性 → 未知机制 → 假说或问题 → 实验设计 → 证据链 → 模型 → 局限和下一步实验

### 方法/AI/工具论文 → problem-to-solution
当前瓶颈 → 提出方法 → 工作流或架构 → 评估设计 → 与基线对比的性能 → 消融/稳健性/失败案例 → 复用场景和局限

### 资源/数据集/图谱/基准论文 → workflow-to-validation
为什么需要此资源 → 数据集/队列/样本设计 → 生成和质量控制工作流 → 主全景图 → 验证和可复现性 → 示例生物学或技术洞察 → 访问/复用/边界

### 临床/人群/干预研究 → design-to-inference
临床/公共卫生问题 → 研究问题 → 队列/试验/设计 → 终点和变量 → 主要结果 → 亚组/敏感性/二次分析 → 偏倚/局限/实践含义

## 输出文件

### 必需
1. `output/final_presentation_cn.pptx` — 主要交付物：完整中文 PPTX
2. `output/qa_report.md` — 简短质量报告：PPTX状态、幻灯片数、嵌入图表、遗漏/占位图表、验证方法、已知局限

### 按需
- `output/ppt_outline_cn.md` — 中文大纲（论文信息、类型、中心论点、幻灯片结构）
- `output/figure_plan.md` — 图表选择计划
- `output/assets/figures/` — 提取/裁切的图表素材

## 质量规则

- 工具有就构建 `.pptx`
- 不停在 Markdown 大纲或脚本
- 不编造结果、方法、数字或图表细节
- 不用文字堆满幻灯片
- 图表可用时不使结果幻灯片纯文字
- 每张幻灯片为论文论证服务
- 确保图表在演示尺度下可读
- 确保文字/图注/图表不重叠
- 不确知和缺失源材料须明确记录
